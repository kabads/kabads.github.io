1 Deep Dive

Sample String: 

fooaaaabar
fooabar
foobar
fooaabar
fooxxxbar
fooxbar

We want to match the first four - so if it has an 'x' in it, we don't want. If it has an 'a' or not, we can accept it.

We want to accept 'foo | any number of 'a's (or not) | bar'.

* is the first symbol. a* means accept 0 or more occurences of the letter 'a'.

fooa*bar is the solution.

2 The Basic Set

2.1 The Wildcard Symbol

fooabar
fooxbar
baryfoo
foobar
fooxybar
foocbar

We want to match 1, 2 and 6.

The letter between foo and bar is unpredictable. But the frequency is the same (one letter in between).

The solution is 'foo.bar'. The . symbol can allow any one character.

grep 'foo.bar' regex.sample1

2.2 Wildcard Asterisk Combo

foobar
barfoo
fooabcbar
boobxcbar
barcbyfoo
foozbar
barafoo
barabfoo

We want to accept 1, 3, 4 and 6. This means any number of any letters.

.* is zero or more occurrences of any letters.

2.3 Representing Whitespaces

fooxxxbar
foo bar
fooxbar
fooxxbar
foo bar
foo    bar
foobar
fooyyybar

We want to accept anything that has any number of spaces and 0 spaces, but nothing else.

\s represents one whitespace.

\s* represents zero or more occurences of spaces. 

grep 'foo\s*bar' regex.sample3

2.4 Character Classes

foo
moo
coo
doo
poo
loo
boo
hoo

We want to accept foo, coo and loo.

We can create a character class with: [charsherer]. It is only one character position. It is not a wildcard.

grep '[fcl]oo' regex.sample4

Now we want to exclude moo, and hoo.

grep '[fcdplb]oo' regex.sample4 is the solution. As m and h are not included in the class, they are excluded.

However, if there are too many characters in the class, it becomes unmanageable.

So, how can we exclude 'm' and 'h'?

grep '[^mh]oo' regex.sample4 is the solution. The ^ denotes anything other than these characters.

Now we are excluding characters.

2.6 Escaping with Backslash

xxx.yy
xx.yyyy
x.y
xy
xxyy
yyxx
yx
yxxx

We want xxx.yy, xx.yyyy, x.yy. They all have the full-stop in common.

How do we reference the full-stop without it being seen as a wildcard?

There are special characters that mean something in regx ^, $, ., [, (, ), \.

When we backslash a character, we escape the character after it and it is passed to be a normal (literal) character.

Solution: grep 'x*\.y*' regex.sample 

x#y
x:y
x.y
x&y
x%y

We want to accept x#y, x:y and x.y.

If a '.' is inside [ ], it does'nt need to be escaped.

Solution:

grep 'x[#:.]y' regex.sample 

x#y
x:y
x^y 
x&y
x%y

We want to find x#y, x:y, and xy.

Solution: grep 'x[#:\^]y' regex.sample 

x#y
x\y
x^y
x&y
x%y

We want to find x#, x\y, xy.

Solution: grep 'x[#\\\^]y' regex.sample

2.7 Anchors

foo bar baz
bar foo baz
baz foo bar
bar baz foo
foo baz bar
baz bar foo

This is a combination of three words in different order.

We want to search for foo bar baz, and foo baz bar.

foo | bar baz foo | baz bar

foo is a fixed string. The latter part is variable. .* will not work here as some of the other entries will also be included.

Only the strings that begin with foo will qualify. This brings us to the new character ^ which means that something at the beginning of the line - and nowhere else. Caret is also a negation symbol, if it is in the square brackets [^ ]. This means negation. If outside, it is interpreted as the place holder for the beginning of the line.

grep '^foo.*' regex.sample

Next example: 

foo bar baz
bar foo baz
baz foo bar
bar baz foo
foo baz bar
baz bar foo

We want to accept 'baz foo bar' and 'foo baz bar'. They both end in bar. This brings in the '$' character. This is the placeholder for the end of the line.

Solution:

grep .*bar$' regex.sample

Next example. 

foo
foo bar
baz foo
foo bar baz
baz bar foo

I only want to accept only 'foo'. We can combine the anchors for this:

Solution: grep '^foo$' regex.sample

3 The Extended Set

The extended set comes with more symbols:

Symbol What does it represent?
+ One or more occurrences of the character that precedes this + symbol
? Zero or one occurence of the character that precedes this question mark
pat1‚ùòpat2 Matches either the pattern pat1 or the pattern pat2
( ) Divides patterns into groups
{m} Exactly 'm' occurences of whatever precedes
{m,n} At least m and n occurences of whatever precedes. Only one of m, n is mandatory. Other can be left blank

3.1 Curly Braces Repeater

834
519
4874
5
89
45687
25
645

We want to select 834, 519, 645. These are all three digit numbers. That is the pattern.

This is how it could be represented: 

^[0-9][0-9][0-9]$

This is duplication though. What if it were 8 numbers. This doesn't scale.

Solution: 

grep '^[0-9]{3}$' regex.sample

Note that we have to have the anchors as some of these numbers are more than 3 digits, which means they would still match. The anchors prevent this.

Next example: 

lion
tiger
leopard
fox
kangaroo
bat
mouse
cuckoo
deer

We want lion, tiger, mouse, cuckoo, deer. We have 4, 5 and 6 letter words.

This could be represented as this: 

^[a-z]{4}$
^[a-z]{5}$
^[a-z]{6}$

Again, this does not scale, so we can represent a range of occurrences: 

^[a-z]{4,6}$

3.2 Single Ended Curly Braces Repeater

ha
hahahahaha
hahaha
hahahaha
haha

hahahahahaha
hahahahahahahaha
hahahahahahahahaha

We want to accept, hahahahaha, hahahahahaha, hahahahahahahaha and hahahahahahahahaha.

This is a range of repetitions from 4 to 9.

Solution: 

grep '(ha){4,}' regex.sample

The lack of the second digit after 4 means that this can go on infinitely to match (i.e. more than 9 occurrences).

Next example: 

ha
haha
hahahahaha
hahahaha
hahaha
hahahahahahaha
hahahahahaha

I want to search for ha and haha only.

We have used the parenthesis to create a group of characters.

We can negate the repetition statement.

Solution: 

grep '^(ha){,2}$' regex.sample
